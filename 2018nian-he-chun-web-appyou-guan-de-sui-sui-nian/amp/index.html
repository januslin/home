2018年和纯Web App有关的碎碎念

by Hui Zhong

2018-06-04

琐事缠身小半年，2018年也过去了四分之一，很久没有更新专栏，向大家道歉——对不起我偷懒了（逃）。

我司前端用Angular.js/Angular/ionic一把梭做项目也有一段时间了，各种业务，从最基本的仓库管理系统、Web IM、物联网的集控系统和市面上看到的绝大多数应用，基本都有做过（毕竟是一个外包公司）。做得越久越发现，我们做的事情慢慢的会从Hybrid App到纯Web App过度，为什么这么说？现在Web的解决方案越来越多了，即使他是experimental的解决方案。

从最早2016年开始，我用Angular.js做Web IM，那时候我还不知道RX.js，写起来要多痛苦有多痛苦。现在，我觉得拿Rx.js重新写一遍，代码应该漂亮很多。

玩过一段时间的WebRTC，写了一个项目在GitHub上，思考了如何用P2P和WebRTC来做直播解决方案。

项目越来越多，时间也越来越少，QR-Code-Scanner、Push、Application Upgrader、Image-Cropper（其实我写过一套Angular.js的，没有移植到Angular上）都直接去Google找现成可兼容的稳定解决方案或者服务提供商，等我回过头来思考我接下来要怎么做事的时候，我突然觉得，是不是可以慢慢的放弃一些Native Plugin了呢？

一个普通的应用，除了一些标准的业务逻辑，比如数据的CRUD和交互之外，还有一些之前Web（我认为）实现不了的东西，貌似都有了Web的解决方案了。比如：

  1. 拍照、录像、录音、扫码：WebRTC和背后的Stream API等等
  2. 离线能力、推送、位置服务：Service Work、Notification API、Geolocation API
  3. 游戏：egret等引擎
  4. AR：ARCore、ARKit实现
  5. 电话、短信、普通的下载：a标签下的href
  6. 蓝牙、NFC：有experimental API
  7. 本地文件：File相关的Web API或许可以

细细思考一下，哪些业务没办法用纯Web的方式实现：

  1. 联系人、通讯录、通话历史记录
  2. 本地的多媒体播放器
  3. 需要蓝牙、NFC（用Web存在兼容性问题）
  4. 各种手机管家（抱歉我真没开发过）
  5. 桌面背景、系统美化（真的跟Web没什么关系）
  6. VPN（这还用说吗）

而我们大多数情况的App是怎么样的呢？

  1. 视频、音乐、漫画
  2. 聊天、直播
  3. 游戏（其实egret做大型Web游戏还要有一段距离）
  4. 购物、共享租赁、旅行
  5. 美颜相机
  6. 新闻、咨询订阅（微博也算吧）
  7. 笔记、词典（其实用的应该不多）

那这么看来，是不是大部分的应用的确似乎不需要安装就能click and run才对。

今年，这个专栏的基调大概就是这样了吧。
