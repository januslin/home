设计模式ECMAScript实现：二、从单例模式到控制反转

单例模式

单例模式在许多地方都会用到，比如你需要使用一个全局统一的数据、或者一个库，不需要将其多次实例化到内存中。我们来看一下最基本的一种单例模式：

示例

let moduleA = (function(){
    let instance = undefined;
    
    class A {
        constructor() {
            this.name = "A";
        }
    }
    
    return {
        getInstance : () => {
            if(!instance) {
                instance = new A();
            }
            return instance;
        }
    }
})();

通常我们可以通过这种方式去定义一个全局唯一的service\provider（比如负责提供某一组Ajax请求的一个对象，就是一个service）。

控制反转(Inversion of Control)

很多时候，我们在组装一个完整的数据需要显式的去组装，比如我是一个厨师，我需要一个苹果和一个橘子来榨汁，那我们一般会怎么做呢？

示例

class Apple {
    constructor() {
        this.name = "apple";
    }
    
    getApple() {
        return this.name;
    }
}
    
class Orange {
    constructor() {
        this.name = "orange";
    }
    
    getOrange() {
        return this.name;
    }
}
    
class Cooker {
    constructor(apple,orange) {
        this.apple = apple;
        this.orange = orange;
    }
    
}
    
new Cooker(new Apple(), new Orange());

这时候你会发现，厨师既要关心苹果，又要关心橘子，真是太麻烦了，为什么不找一个配菜员帮你解决这个事情呢？如果在厨师需要一个苹果的时候，配菜员主动给个苹果，那就好了。

我们将数据生产的控制交给一个专门的角色去做，而用户只在需要某个数据的时候叫那个角色提供，这种模式就叫做“控制反转”。而控制反转模式的一种实现就叫做“依赖注入(Dependency Injection)”。

实现一个简单的依赖注入

我们在使用依赖注入的时候通常需要定义一个可以被依赖注入的类型，以及我们在使用一个模块时候需要通过声明的方式来引用这些被注入的对象。

示例

let injector = (function(){
    let instance = undefined;
    
    class Injector {
        constructor() {
            this.inject = {}
        }
        
        define(name, func) {
            // TODO exception handler is needed
            this.inject[name] = func();
        }
        
        use(injectList, func) {
            // TODO exception handler is needed
            let injectable = [];
            injectList.map((name) => {
                injectable.push(this.inject[name])
            })
            func.apply(this, injectable);
        }
    }
    
    return {
        getInstance : () => {
            if(!instance) {
                instance = new Injector();
            }
            return instance;
        }
    }
})();
    
// 使用依赖注入
// 代码中还有一部分没有完成，就是对入参的判断（这时候TypeScript的好就体现出来了lol），以及可以被注入的数据相互依赖的实现。

injector.getInstance().define("Apple", () => {
    class Apple {
        constructor() {
            this.name = "apple";
        }
        
        getApple() {
            return this.name;
        }
    }
    return new Apple();
})
    
injector.getInstance().define("Orange", () => {
    class Orange {
        constructor() {
            this.name = "orange";
        }
        
        getOrange() {
            return this.name;
        }
    }
    return new Orange();
})
    
injector.getInstance().use(["Apple", "Orange"], (apple,orange) => {
    console.log(apple.getApple()); // 'apple'
    console.log(orange.getOrange()); // 'orange'
})

这种形式其实类似于require.js，我们将模块分两种，一种是我们需要依赖的，一种是我们需要实现的业务逻辑。然而这只是一个小小的演示代码，Ng\vue等前端框架也大量的采用了依赖注入来帮助用户维护代码，有兴趣的同学也可以仔细读读大神们的代码。
